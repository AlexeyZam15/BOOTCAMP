# Шейкерная (коктейльная) сортировка
## Описание алгоритма
```
Производится многократный прогон по массиву, соседние элементы сравниваются и, если левый элемент в паре больше правого, меняются местами. 
При достижении конца массива направление меняется на противоположное, рабочая область уменьшается на 1 элемент с конца. 
При достижении начала массива направление опять меняется на противоположное, рабочая область уменьшается на 1 элемент с начала. 
Алгоритм повторяется пока в рабочей области не останется 1 элемент.
```
## Пример работы алгоритма
### Начальный массив: [3,1,5,0,7,9,8]
```
1.
Рабочая область [3,1,5,0,7,9,8]
[3, 1, 5, 0, 7, 9, 8]| 3>1  | 3 меняется с 1
[1, 3, 5, 0, 7, 9, 8]| 3<=5 | 3 не меняется с 5
[1, 3, 5, 0, 7, 9, 8]| 5>0  | 5 меняется с 0
[1, 3, 0, 5, 7, 9, 8]| 5<=7 | 5 не меняется с 7
[1, 3, 0, 5, 7, 9, 8]| 7<=9 | 7 не меняется с 9
[1, 3, 0, 5, 7, 9, 8]| 9>8  | 9 меняется с 8
Рабочая область [1,3,0,5,7,8]
смена направления
[1, 3, 0, 5, 7, 8]| 7<=8 | 7 не меняется с 8
[1, 3, 0, 5, 7, 8]| 5<=7 | 5 не меняется с 7
[1, 3, 0, 5, 7, 8]| 0<=5 | 0 не меняется с 5
[1, 3, 0, 5, 7, 8]| 3>0  | 3 меняется с 0
[1, 0, 3, 5, 7, 8]| 1>0  | 1 меняется с 0
2.
Рабочая область [1,3,5,7,8]
[1, 3, 5, 7, 8]| 1<=3 | 1 не меняется с 3
[1, 3, 5, 7, 8]| 3<=5 | 3 не меняется с 5
[1, 3, 5, 7, 8]| 5<=7 | 5 не меняется с 7
[1, 3, 5, 7, 8]| 7<=8 | 7 не меняется с 8
Рабочая область [1,3,5,7]
смена направления
[1, 3, 5, 7]| 5<=7 | 5 не меняется с 7
[1, 3, 5, 7]| 3<=5 | 3 не меняется с 5
[1, 3, 5, 7]| 1<=3 | 1 не меняется с 3
3.
Рабочая область [3,5,7]
[3, 5, 7]| 3<=5 | 3 не меняется с 5
[3, 5, 7]| 5<=7 | 5 не меняется с 7
Рабочая область [3,5]
смена направления
[3, 5]| 3<=5 | 3 не меняется с 5
```
### Конечный массив:  [0,1,3,5,7,8,9]
